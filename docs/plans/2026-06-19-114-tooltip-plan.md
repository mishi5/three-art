# #114 ノード/param/ポートのマウスオーバー説明ツールチップ

対象 Issue: https://github.com/mishi5/three-art/issues/114
関連: Epic #56 / #92（ワークスペースのズーム — 座標追従の考慮）

## 目的

node-vj エディタで、ノード名・各パラメータ・ポートにマウスオーバーすると
ツールチップでその説明が表示されるようにする。各ノード・param が何をするものか
分かりやすくし、エディタの学習コストを下げる。

## 設計方針

### データモデル（`graph/node-type.ts`）
- `NodeTypeDef` / `ParamDef` / `PortDef` に任意の `description?: string` を追加。
- 未記入のものはツールチップ非表示（任意フィールド）。段階的に拡充できる。

### 表示方式: canvas 直描画
- 既存エディタは Canvas2D 一枚で完結しており、DOM オーバーレイを増やさない方が一貫する。
- ツールチップは offset 変換の**外**（スクリーン座標）で最前面に描く。
  → ズーム（#92）が入っても座標追従の影響を受けにくい（カーソルのスクリーン座標基準）。

### ホバー検出
- 既存の統一ヒットテスト `editor/hit-test.ts` を流用（port / param / node を遮蔽つきで解決）。
- ドラッグ中は非表示。一定時間（`TOOLTIP_DELAY_MS = 450ms`）ホバー継続で表示。
- 別対象へ移ったら表示タイマーを計り直す。

### 純関数（`editor/tooltip.ts`・TDD 対象）
- `tooltipForHit(hit, registry)`: ヒット結果 → `{ title, body }`。説明が無ければ null。
  - node: type を見出し / param: ラベルを見出し / port: ポートラベル（param 入力は params から解決）。
- `tooltipBox(...)`: カーソル右下に出し、画面端で左/上へ反転、最終クランプ。
- `wrapLines(text, maxWidth, measure)`: 本文を最大幅で折り返し。measure を注入して測定をテスト可能に。

### 描画統合（`editor/NodeEditor.ts`）
- onMove でホバー状態を更新（`updateHover`）、draw() 末尾で `drawTooltip()`。
- 本文は `MAX_W=280` で折り返し、タイトル+本文行数から箱サイズを算出。

### 説明文の付与
- 各ノード定義（`nodes/*.ts`）に `description` を記述。一次情報源は各ファイルの JSDoc。
- 共有定義 `ONSET_PARAMS` / `AUDIO_FEATURE_OUTPUTS`（`audio-feature-logic.ts`）にも付与し、
  Mic/Display/AudioFile 入力へ一括反映。

## テスト
- `editor/tooltip.test.ts`: tooltipForHit（node/param/port/null/未登録）・tooltipBox（端回避）・wrapLines。

## 動作確認
- dev サーバ（`bun run dev:vj`・port 3000）でノード/param/ポートにマウスオーバーし、
  約 0.45 秒で説明ツールチップが出ること、画面端で反転すること、ドラッグ中は出ないことを確認。
