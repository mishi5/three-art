# #149 tex 歪みエフェクトノードを追加

対象 Issue: https://github.com/mishi5/three-art/issues/149
親 Epic: #56

## 目的
texture の UV を変形して再サンプルする歪み系エフェクトを追加する（魚眼/逆歪み・マッピング・ねじれ）。

## 設計
- 新ノード `DistortNode`（category: `effect`, isSink: true, `in` texture → `texture`）。
  Kaleidoscope/Fractal と同じ `ShaderSurface`/`NDC_VERTEX`/`EFFECT_ENABLED_PARAM`/bypass パターン。
- mode（enum）で歪みの種類を切替:
  - `fisheye`: 半径方向スケール。**amount の符号で 魚眼(>0=中心拡大) / 逆歪み(<0=ピンクッション)** を兼ねる。
  - `twist`: 中心まわりの渦巻き（距離で減衰）= ねじれ。
  - `wave`: 正弦波 UV リップル（radius で波長）= マッピング的な座標変形。
- param: enabled / mode / amount(-1..1) / centerX / centerY / radius / mix。
  歪み量・中心・半径を param 化し、他ノード（audio 等）から駆動可能（数値 param 入力）。
- `distortModeInt(mode)` 純関数（fisheye=0/twist=1/wave=2, 未知=0）。
- uAspect は renderer サイズから算出（中心からの距離をアスペクト補正）。

## テスト
- `distort.test.ts`: ポート（texture→texture）・category・no-op・params・mode enum・
  amount 符号域・distortModeInt・registry 登録。全749件パス。
- GLSL は Playwright スモークで TextureGenerator(グラデ)→Distort(twist)→Screen の
  コンパイル＆描画（渦巻き）を確認。

## 成果物
- `nodes/DistortNode.ts`（新規）・registry 登録・テスト。

## 備考
- 「マッピング（任意の座標変換）」は wave モード＋中心/半径/量の組合せで表現。
  より高度なリマップ（極座標等）は将来の追加モードで拡張可能。
