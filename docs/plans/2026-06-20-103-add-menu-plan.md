# #103 ノード追加メニューの整理 + 右クリックメニュー

対象 Issue: https://github.com/mishi5/three-art/issues/103
関連: Epic #56

## 目的

ノード追加の操作性改善。左上ツールバーをカテゴリ別に整理し、キャンバス右クリックの
コンテキストメニュー（クリック位置に追加 / ノード操作）を実装する。

## 設計

### 純関数 `editor/node-menu.ts`（TDD）
- `CATEGORY_ORDER = [input, process, visual, effect, output]`、未知/未設定は末尾 `other`。
- `groupNodesByCategory(defs)`: カテゴリ順にグルーピング（空カテゴリは除外、カテゴリ内は登録順維持）。

### ツールバー整理（buildToolbar）
- 全 `+ 型` ボタンをカテゴリ別の小グループ（ラベル付き）に並べ替え。デバッグトグル・ヒントは維持。

### 右クリックメニュー（DOM）
- `contextmenu` はネイティブ抑止のみ（従来どおり）。実メニューは `onUp` で出す。
- 右ボタン押下は従来どおりパン開始。`onUp` で **右ボタン かつ 移動量 < しきい値** ならメニューを開く
  （右ドラッグはパンのまま）。
- 空白上 → カテゴリ別の追加メニュー。選んだ型を**右クリックした world 位置**に生成（`addNodeOfType(type, worldPos)`）。
- ノード上 → 複製 / 削除（対象が未選択なら単独選択化。複数選択時は件数表示）。
- メニュー外クリック（capture pointerdown）で閉じる。`dispose` でも閉じる。

### addNodeOfType
- 第2引数 `worldPos?` を追加。未指定（ツールバー）は従来の画面左上付近、指定（右クリック）はその位置。

## テスト
- `editor/node-menu.test.ts`: カテゴリ順・登録順維持・空カテゴリ除外・other 集約。
- DOM メニュー/位置生成は Playwright スモークで確認（空白右クリック→追加→クリック位置生成）。

## 動作確認
- 左上ツールバーがカテゴリ別に並ぶ。
- 空白右クリック→メニュー→型選択でその位置にノード生成。
- ノード右クリック→複製/削除。右ドラッグはパン、メニュー外クリックで閉じる。
