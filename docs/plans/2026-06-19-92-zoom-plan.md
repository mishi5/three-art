# #92 ワークスペースの拡大縮小（ズーム）

対象 Issue: https://github.com/mishi5/three-art/issues/92
関連: Epic #56 / #114（ツールチップ — スクリーン座標描画で影響なし）

## 目的

ノード増加時に全体俯瞰・細部操作ができるよう、node-vj エディタのキャンバスに
ズーム（拡大縮小）を追加する。

## 座標モデル

`screen = world * scale + offset`（offset はスクリーン px のパン量、scale は拡大率）。
これまで `world = screen - offset`（scale=1 固定相当）だったところに scale を導入。

## 設計

### 純関数 `editor/viewport.ts`（TDD 対象）
- `MIN_SCALE=0.25` / `MAX_SCALE=2.0` / `clampScale`
- `screenToWorld(sx,sy,offset,scale)` / `worldToScreen(wx,wy,offset,scale)`
- `zoomAt(sx,sy,offset,scale,factor)`: カーソル下の world 点を画面上で固定したまま
  factor 倍ズーム（scale クランプ。クランプ時もカーソル点を保持）。

### NodeEditor 統合
- `scale` フィールド追加。
- `toWorld` を `screenToWorld` に置換（全ヒットテスト/ドラッグ/配線が world 基準なので追従）。
- draw: `translate(offset)` の後に `scale(scale)`（dpr → translate → scale の順）。
- ノード追加位置: 画面 120px 付近を `screenToWorld` で world 化。
- param オーバーレイ（DOM）: `worldToScreen` で位置、幅・フォントを `scale` 倍にして
  canvas の param 行へ重なるよう追従（`param-overlay.ts` に `fontPx` を追加）。
- 操作:
  - ホイール/トラックパッドのピンチ（ctrl+wheel）でカーソル中心ズーム（`onWheel`、passive:false）。
  - `0` キーでズーム 100%（画面中心を固定して戻す）。
- ツールチップ（#114）はスクリーン座標で最前面描画のためズーム非依存（ズーム中は消す）。
- プレビュー小窓は canvas 内描画なので scale で自然に追従。

## テスト
- `editor/viewport.test.ts`: clampScale / 往復変換 / zoomAt のカーソル固定・クランプ。
- ヒットテスト等は world 基準で不変のため既存テストで担保。
- 実ホイール/DOM 追従は手動確認。

## 動作確認
- ホイールでカーソル中心に拡大縮小、上限/下限で頭打ち、`0` で 100% 復帰。
- ズーム中もノード移動・配線・param 編集（オーバーレイ位置）が正しく動く。
