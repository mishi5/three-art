# #157 マスク/キーイングノード（クロマキー・ルマキー）

対象 Issue: https://github.com/mishi5/three-art/issues/157
親 Epic: #56

## 目的
texture を合成する際のマスク/キーイング（クロマキー・ルマキー）を追加する。

## 設計
- 新ノード `KeyNode`（category: `visual`, isSink: true）。Blend と同じ 2-texture コンポジタの括り。
  - 入力: `fg`（前景・キーイング対象）/ `bg`（背景・抜いた所に出る）。出力: `texture`。
  - 既存 `ShaderSurface`/`NDC_VERTEX`/`blackTexture` を流用（未接続入力は黒）。
- mode（enum）:
  - `chroma`: キー色 `uKey`(RGB) からの距離で透過。threshold/softness でエッジ、
    spill でエッジのキー色かぶりを輝度グレーへ寄せて抑制。
  - `luma`: 輝度がしきい値以上を残す（threshold/softness）。
- `invert`(on/off) で透過する側を反転。
- FRAG: `keep`（1=fg, 0=bg）を算出し `mix(bg, fg, keep)`。
- param: mode / keyR / keyG / keyB / threshold / softness / spill / invert。数値 param は他ノード駆動可。
- `keyModeInt(mode)` 純関数（chroma=0/luma=1, 未知=0）。

## テスト
- `key.test.ts`: fg/bg 2 入力・texture 出力・mode enum・params・keyModeInt・registry・no-op。
  全757件パス。
- GLSL は Playwright スモークで fg(緑→赤グラデ)+bg(青) のクロマキー合成（緑側が青に置換）を確認。

## 成果物
- `nodes/KeyNode.ts`（新規）・registry 登録（visual セクション）・テスト。

## 備考
- マスク texture 入力による合成は将来の拡張余地（今回は chroma/luma の自動マスク）。
